# 技能：QCoDeS 驱动摘要提取 (Driver Cheatsheet Extraction)
## 触发时机
当用户提供一段冗长的 QCoDeS 测量范例代码，并要求你“提取驱动摘要”时。
## 执行逻辑
你不需要把整个范例代码死记硬背。你需要分析用户提供的代码，提取最核心的 API 调用语法，并提炼成 `memory/driver_cheatsheet.md`。
## 提取规范
1. **初始化部分**：提取 Station 初始化和组件挂载的代码片段。
2. **微波源操作 (Microwave)**：提取如何 set 频率 (frequency)、功率 (power) 和状态 (status 'on'/'off') 的精准一行代码。
3. **直流设备操作 (DC - 重点标注 IV 级权限)**：
   - 提取设置电压/电流的代码。
   - **强制改写**：在摘要中，必须标注：“未来调用此 DC API 时，必须套用 IV 级权限双重确认逻辑，并明确向用户报告具体偏置值。”
4. **测量循环与数据库存储**：提取 `Measurement(exp=...)` 注册参数、`run()` 循环以及向本地数据库存入数据的极简模板。
## 产出目标
生成的 `driver_cheatsheet.md` 必须极度精简，像一本 API 字典。你在未来写所有测量脚本时，都必须仅参考这本字典中的语法。
